# Sticky Notes

基于 Electrobun 和 TypeScript 的轻量 Windows 桌面便签与待办工具。

## 功能

### 便签

- 支持同时创建和管理多个独立便签窗口
- 正文输入后自动保存，隐藏或退出应用前会刷新待保存内容
- 双击编辑便签标题，并同步更新窗口标题和托盘便签列表
- 单独置顶或取消置顶任意便签
- 单独隐藏到托盘或永久删除当前便签；删除最后一个便签后可从托盘重新新建

### 待办

- 每个便签可独立切换便签模式和待办模式
- 支持新增、编辑、删除和完成待办
- 已完成待办自动进入归档，支持恢复、单独删除和清空归档
- 待办模式、未完成列表和归档列表均持久化保存

### 窗口

- 无边框窗口，可拖动标题栏移动
- 支持拖动四条边和四个角调整尺寸，并限制最小尺寸
- 记忆每个便签的位置、尺寸、置顶状态和吸附状态
- 新建便签优先放置在副屏，多窗口自动错位排列
- 支持多显示器工作区识别，避免窗口意外完全移出可见区域
- 靠近屏幕外边缘时自动吸附并收缩，鼠标移入自动展开，移出后延迟收起
- 吸附、收缩和展开带平滑动画；屏幕之间的内部边界不会误触发外边缘吸附

### 系统托盘

- 托盘菜单可新建便签，并按标题打开指定便签
- 支持显示全部和隐藏全部便签
- 窗口关闭操作只影响当前便签，退出整个应用仅保留在托盘菜单
- 托盘退出前统一保存所有便签内容和窗口位置

### 本地存储

- 便签正文、标题、待办、归档、模式、置顶状态和窗口布局统一保存为版本化 JSON
- 写入时使用临时文件替换，数据损坏时自动备份原文件并恢复可用状态
- 支持首次启动时导入旧版 `note.txt` 正文

## 架构

```text
src/
├── bun/          # 数据仓库、窗口、托盘和应用生命周期
├── mainview/     # HTML/CSS/TypeScript 界面
├── shared/       # 主进程与界面共享的 RPC 契约
└── assets/       # 应用资源
```

数据文件为 `notes.json`，保存在 Electrobun 的 `Utils.paths.userData` 目录。Windows 下该目录位于当前用户的 Local AppData。

## 开发

安装依赖后即可运行，无需全局安装 Bun。

```bash
npm install
npm run dev
```

类型检查：

```bash
npm run typecheck
```

## 构建

生成 Windows x64 单文件安装程序：

```bash
npm run package:win
```

安装程序输出为 `artifacts/StickyNotes-Setup-x64.exe`。首次打包需要在项目的 `.tools/inno/compiler/` 目录准备 Inno Setup 7 编译器。

## 快捷键

| 快捷键 | 功能 |
| --- | --- |
| `Ctrl+Alt+N` | 新建便签 |
| `Ctrl+Alt+T` | 切换便签/待办模式 |
| `Esc` | 关闭弹窗、退出待办模式或隐藏窗口 |
